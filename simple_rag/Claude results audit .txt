     Plan: GNEM RAG Codebase Audit + RAGAS Evaluation Implementation

     Context

     This is a two-phase task for a Georgia EV supply-chain RAG system that
     compares three retrieval modes (dense, sparse, hybrid) across 50
     human-validated questions. Phase 1 produces a technical audit report. Phase
      2 implements RAGAS-based evaluation and human-gold comparison scripts.

     ---
     Phase 1 — Audit Findings (pre-determined from codebase read)

     Verdict: Valid with minor fixes

     What Is Correct:
     - Dense: ChromaDB + nomic-embed-text via Ollama ✓
     - Sparse: real BM25Okapi (rank_bm25), genuinely separate ✓
     - Hybrid: true weighted combination (alpha=0.6 vector, 0.4 BM25) ✓
     - Same 50 questions, same LLM (gemma3:27b), same prompt for all 3 modes ✓
     - question_id Q001-Q050 preserved across all output sheets ✓
     - top_k=80, pool=250 consistent; outputs timestamped, never overwritten ✓
     - Evidence metadata fully preserved in Context_Dense/Sparse/Hybrid sheets ✓
     - Prompt enforces KB-only answers (no golden answers used during
     generation) ✓
     - Per-question error handling (failed rows get error column, not silently
     dropped) ✓

     Problems Found:
     1. Parent-child chunking is in name only: 1 row = 1 chunk, parent_id = same
      chunk's record_id. No actual child decomposition. parent_text =
     embedding_text of same row. Misleading architecture claim.
     2. BM25 index not persisted: Rebuilt in-memory on every run — O(n)
     ingestion overhead at startup.
     3. Vector normalization formula wrong for cosine: 1/(1+dist) is designed
     for L2 distances. ChromaDB cosine distance ∈ [0,2] → this maps to [0.33,1]
     not [0,1]. Correct formula: 1 - dist/2. Min-max normalisation mitigates but
      scores ≠ true cosine similarity.
     4. Sparse mode calls embed_query unnecessarily: query embedded even when
     only BM25 used (performance waste, not correctness error).
     5. questions.xlsx lacks question_id column: IDs auto-generated Q001-Q050 by
      row order. Fragile if rows ever shuffle.
     6. Context_Sparse has 3949 rows vs 4000 for Dense/Hybrid: BM25 returned <
     80 results for some questions. Expected but worth documenting.
     7. SYSTEM_PROMPT has decorative {question}/{context} placeholders: Never
     filled in (passed as static string, not formatted). Confusing but not a
     bug.
     8. ragas missing from requirements.txt.

     ---
     Phase 2 — Implementation

     Key Architecture Decisions

     Source file for evaluation:
     - Primary: simple_rag/outputs/generated_answers/generated_answers_current_c

     - Sheets: RAG_Dense, Context_Dense, RAG_Sparse, Context_Sparse, RAG_Hybrid,

     Context extraction (critical):
     - Do NOT parse the retrieved_context text blob in RAG_* sheets
     - Use the text column in Context_* sheets, group by question_id → list of
     strings
     - Context_Sparse has 3949 rows (some questions have < 80 contexts); handle
